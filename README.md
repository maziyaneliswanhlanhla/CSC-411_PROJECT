
PRODUCERCONSUMERPROBLEM
CSC411 – Producer Consumer Mini Project

Overview
This project implements the **Producer–Consumer problem** using semaphores, XML, and socket programming.  
Developed as part of the **Integrative Programming Technologies (CSC411)** course, University of Eswatini.

Features
- Producer generates random IT student data and saves it as XML files.
- Consumer unwraps XML, calculates averages, and determines pass/fail.
- Shared buffer (max 10 elements) using semaphores for synchronization.
- Socket implementation for distributed communication.
- GitHub used for collaboration and version control.
